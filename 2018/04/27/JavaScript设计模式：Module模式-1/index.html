Posted on 2018-04-27 | In JavaScript

模块模式

模块是任何强大应用程序架构中不可或缺的一部分，它可以帮助我们清晰的分离和组织项目中的代码单元

在JavaScript中，有几种实现模块的方法，包括

·对象字面量表示法
·Module模块
·AMD模块
·CommonJS模块
·ECMAScript6模块

Module模块

Module模式最初被定义为一种在传统软件工程中为类提供私有和公有封装的方法。
在JavaScript中，Module模式用于进一步模拟类的概念，通过这种方式，能够使一个单独的对象拥有共有/私有方法和变量，从而屏蔽来自全局作用域的特殊部分。这样使得函数名与页面上其他脚本定义的函数冲突可能性降低。

特点：私有

Module模式使用闭包封装“私有”状态和组织。它提供了一种包装混合共有/私有方法和变量的方式，防止其泄露至全局作用域，减少冲突。通过该模式，旺旺只要返回一个公有API，而其他的一切则都维持在私有闭包里。
该模式利用IIFE返回一个对象。
在JavaScript中没意义真正意义上的“私有”，不像Java等传统语言，JavaScript没有访问修饰符，私有的实质是利用函数作用域模拟了私有的概念。在Module模式中，由于闭包的存在，声明的变量和方法只在该模式内部可用。但在返回对象上定义的变量和方法，则对外部使用者都是可用的。

可以看到我在立即调用函数内部定义了privateName变量，然后返回了一个包含两个方法和一个属性的对象，将返回值赋给顶层作用域的变量名myModule，通过这两个方法可以对函数作用域内部的privateName进行操作，但是却不能直接利用点表示法“myModule.privateName”访问这个属性，这就像私有变量，因为privateName的存在被局限在了模块的闭包内，因此只能通过返回的对象的两个方法对其操作。而返回对象的属性则可以通过点表示法“myModule.publicAge”直接读取，而它是作为公有变量的存在。
